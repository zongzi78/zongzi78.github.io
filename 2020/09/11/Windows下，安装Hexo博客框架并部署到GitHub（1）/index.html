Windows下，安装Hexo博客框架并部署到GitHub（1）

Posted on 2020-09-11 Edited on 2020-10-01

前言

8月30号就搞定了blog的本地安装与GitHub部署，拖了半个多月才想起要把部署过程写出来，在写之前，首先要感谢B站的up主CodeSheep，是在他的系列blog搭建视频的帮助下才有的这篇文章。 由于安装Node.js和Git，本地安装hexo以及部署到云端的整个流程较长，预计分为两次来写，本篇blog主要写Node.js和Git的安装。

环境配置

Node.js安装

首先进入nodejs的官方网站

可以看到下方的下载按钮，左侧为LTS(Long Term Support)版本，右侧为最新版本，这里下载LTS版本即可。下载完成后可以得到一个名为“node-v12.8.4-x64.msi”的安装包文件。

由于nodejs官网服务器在国外，下载速度比较慢，这里提供一个阿里淘宝的镜像下载地址，里面下载的文件与官网完全一样，请大家放心下载。

下载完成后，双击安装到任意位置，一路默认即可。这里不推荐安装到C盘，可以参照图中我的安装路径。

安装完毕后，使用“win+r”调出Windows的运行界面，输入cmd点击回车，进入命令行界面，输入node -v，若出现“v12.8.4”字样，说明nodejs安装成功，接着输入npm -v，若出现“6.14.6”1，说明npm安装成功。

npm一般是跟nodejs主程序一起自动安装的，至于npm的含义与作用将在后文解释，这样就可以进行下一步操作了。

Git安装

首先进入Git的官方网站，接着点击download按钮，进入下载界面。

按照自己使用的系统下载安装包，这里选择Windows系统，点击Windows图标之后，一般会自动开始下载名为“Git-2.28.0-64-bit.exe”的文件，这是64位系统的安装包。如果你的系统是32位，你可以取消默认的下载进程，在如下图所示的列表里下载32位的安装包。

上面两个链接是安装包，下面两个链接是便携版，这里推荐下载上面的版本。与nodejs一样，git官网服务器在国外，下载速度比较慢，这里提供一个阿里淘宝的镜像下载地址，里面下载的文件与官网完全一样，请大家放心下载。

下载完成后，双击进行安装，这里还是不建议安装到C盘，安装位置可以参照我的

在这一步选择自己喜欢的文本编辑器，我选择的是VS Code

后面没有需要注意的地方了，一路默认即可。安装完成后，桌面上将出现“Git Bash”快捷方式，双击就能进入Git的命令行界面了，由于在Git的安装过程中环境变量都已自动配置好，所以也可以直接进入cmd界面进行Git的一些操作。进入cmd界面，输入git --version，若出现“git version 2.28.0.windows.1”字样，说明安装成功。

npm换源

npm介绍

所谓npm，通俗理解为包管理器，这里的包就是基于nodejs而开发的的一个个插件/模块，而npm就是对这些包进行安装，卸载等操作的管理器。根据菜鸟教程上面的介绍，npm的主要功能有：

  • 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

现在的nodejs已经集成了npm，所以在安装nodejs的时候npm也会随之安装。

建立空白博客框架

首先在计算机任意位置建立一个名为blog的空文件夹，这里我们选择~/blog2

进入cmd命令行界面，左侧的地址为当前的路径地址，输入cd blog将其变更到上一步建立的blog文件夹下。

可以看到现在左侧的地址由C:\Users\xxx\变成了C:\Users\xxx\blog\，这一步是很重要的，它确保了跟你个人博客相关的文件，包括配置文件、博客原文、皮肤文件等等全部位于“C:”这一地址下，便于你以后管理。在cmd终端中输入hexo init安装博客框架，安装完成后，打开“C:”，可以看到里面已经多了很多文件，这些文件就是你的个人博客框架，关于这些文件各自的作用与含义，hexo官网的文档界面的建站部分解释的很清楚，可以自行查看。

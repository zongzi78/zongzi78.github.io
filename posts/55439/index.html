Windows 下，安装 Hexo 博客框架并部署到 GitHub（1）

Posted on 2020-09-11 Edited on 2022-10-01

前言

8 月 30 号就搞定了 blog 的本地安装与 GitHub 部署，拖了半个多月才想起要把部署过程写出来，在写之前，首先要感谢B站的 up 主 CodeSheep，是在他的系列 blog 搭建视频的帮助下才有的这篇文章。
由于安装 Node.js 和 Git，本地安装 Hexo 以及部署到云端的整个流程较长，预计分为两次来写，本篇 blog 主要写 Node.js 和 Git 的安装。

环境配置

Node.js 安装

首先进入其官方网站

可以看到下方的下载按钮，左侧为 LTS（Long Term Support）版本，右侧为最新版本，这里下载 LTS 版本即可。下载完成后可以得到一个名为“node-v12.8.4-x64.msi”的安装包文件。

由于其服务器在国外，下载速度比较慢，这里提供一个阿里淘宝的镜像下载地址，里面下载的文件与官网完全一样，请大家放心下载。

下载完成后，双击安装到任意位置，一路默认即可。这里不推荐安装到C盘，可以参照图中我的安装路径。

安装完毕后，使用“win+r”调出 Windows 的运行界面，输入 cmd 点击回车，进入命令行界面，输入 node -v，若出现“v12.8.4”字样，说明安装成功，接着输入 npm -v，若出现“6.14.6”[^注1]，说明 npm 安装成功。

npm 一般是跟 nodejs 主程序一起自动安装的，至于 npm 的含义与作用将在后文解释，这样就可以进行下一步操作了。

[^注1]:这里的 6.14.6 是我自己电脑上安装的 npm 的版本，不同的版本显示的数字不一样，不必追求一模一样。

npm 换源

npm 介绍

所谓 npm，通俗理解为包管理器，这里的包就是基于 nodejs 而开发的的一个个插件/模块，而 npm 就是对这些包进行安装，卸载等操作的管理器。根据菜鸟教程上面的介绍，npm 的主要功能有：

  • 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

现在的 nodejs 已经集成了 npm，所以在安装 nodejs 的时候 npm 也会随之安装。

建立空白博客框架

首先在计算机任意位置建立一个名为 blog 的空文件夹，这里我们选择 ~/blog[^注2]

[^注2]:符号“~”表示个人文件夹的地址，在 Windows 系统中，一般为“C:\Users\xxx”，xxx为你的用户名。这里之所以选择在这里建立 blog 文件夹的原因，是因为一般 cmd 界面默认就是此地址，便于以后操作。

进入 cmd 命令行界面，左侧的地址为当前的路径地址，输入 cd blog 将其变更到上一步建立的 blog 文件夹下。

可以看到现在左侧的地址由 C:\Users\xxx\ 变成了 C:\Users\xxx\blog\，这一步是很重要的，它确保了跟你个人博客相关的文件，包括配置文件、博客原文、皮肤文件等等全部位于“C:\Users\xxx\blog”这一地址下，便于你以后管理。在 cmd 终端中输入 hexo init 安装博客框架，安装完成后，打开“C:\Users\xxx\blog”，可以看到里面已经多了很多文件，这些文件就是你的个人博客框架，关于这些文件各自的作用与含义，Hexo 官网的文档界面的建站部分解释的很清楚，可以自行查看。
